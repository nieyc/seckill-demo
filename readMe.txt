秒杀系统 demo
打包:
mvn clean package -Dmaven.test.skip=true
运行：
java -jar lockDemo-0.0.1-SNAPSHOT.jar

秒杀技术要点：
1：库存不能超卖(分布式锁来实现)
2:大规模请求不能直接打到数据库，层层过滤（html，css,pic ，web请求均需要缓存）
   1：请求数据进入队列，监听队列并且构造一个线程池从队列取数据，可以自动调节访问数据库并发量
   2：秒杀前的用户刷新页面请求从缓存中查询，页面缓存，redis缓存，本实例只考虑后端架构即redis缓存
     redis 缓存击穿和雪崩问题
 3:商品抢购完，请求不在添加到队列
   4：数据库如果撑不住，可以考虑读写分离，主库写，从库读
3:超过支付时间没有支付的，商品需返回库存，更新缓存
  3.1：定时任务扫描
  3.2：延迟队列，此方案可以保障时间精确
4：前端客户如何在最短时间内知道抢购成功？即使后台通过多线程写入数据库，速度仍然很慢，前端可能需要等几十秒才能有结果，这显然是不可接受的
